Dos módulos — uno construye el maestro, el otro lo usa cada mes para identificar pagos

construir_maestro

Aprende orígenes de pagos históricos y construye el maestro de usuarios

construir_maestro/
├── Inputs/
│   ├── Planilla/
│   │   ← 7 planillas históricas
│   └── Usuario id/
│       ← usuarios_id.xlsx
├── Outputs/
│   └── maestro_yape.xlsx
└── main.py
Inputs
Planillas históricas del banco (.xlsx)
usuarios_id.xlsx con mz y lote
Output
maestro_yape.xlsx — orígenes + nivel confianza

motor_matching

Cada mes identifica a qué mz y lote corresponde cada pago Yape recibido

motor_matching/
├── Inputs/
│   ├── Reporte_mes/
│   │   ← reporte banco mes actual
│   └── Maestro/
│       ← copias manual aquí
├── Outputs/
│   ├── pagos_identificados.xlsx
│   └── pendientes.xlsx
└── main.py
Inputs
Reporte banco del mes (.xlsx)
maestro_yape.xlsx (copiado manual)
Outputs
pagos_identificados.xlsx — con mz y lote
pendientes.xlsx — revisión manual
maestro_yape.xlsx se copia manualmente de Outputs → Inputs/Maestro
Flujo mensual
1. Descargar reporte banco → 2. Pegar en Reporte_mes/ → 3. Correr motor_matching → 4. Revisar outputs → 5. Actualizar maestro si hay nuevos
