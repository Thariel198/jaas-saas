Contrato de formato — trazabilidad_reclamos.xlsx

Módulo: 4b_reclamos  |  Ruta: trazabilidad/trazabilidad_reclamos.xlsx  |  Lifecycle: permanente — nunca se borra  |  Hojas: 1 (Trazabilidad)

Propósito

Registro de auditoría permanente.
Acumula todos los reclamos cerrados (RESUELTO o RECHAZADO), mes a mes, sin eliminar filas antiguas. Cada fila contiene el ciclo completo: quién reclamó, qué tipo, qué se decidió, y qué corrección se acordó.

  • Cada vez que main.py detecta un reclamo con ESTADO = RESUELTO o RECHAZADO, lo mueve aquí.
  • Al cerrar, se mergean los datos de resolucion_reclamos_YYYY-MM.xlsx (CAMPO, VALOR_ACTUAL → VALOR_ANTERIOR, VALOR_A_CORREGIR → VALOR_APLICADO).
  • El archivo crece indefinidamente. No se limita por mes. Las filas nuevas se agregan al final.
  • Nunca se eliminan filas ya escritas — es el historial completo de reclamos cerrados.
Sección "Corrección aplicada" (columnas naranjas):
Registra la corrección acordada — el componente afectado, el valor que tenía antes, y el valor que debería tener.

  • CAMPO: nombre del componente disputado (ej. Convenio, MES ANTERIOR).
  • VALOR_ANTERIOR: valor que tenía DATA_boletas al momento en que se corrió resolucion.py (= VALOR_ACTUAL de resolucion_reclamos).
  • VALOR_APLICADO: valor que el supervisor decidió que debería ser (= VALOR_A_CORREGIR de resolucion_reclamos).
  • Para reclamos RECHAZADO: estas tres columnas quedan vacías (no hubo corrección acordada).
  • Para reclamos cerrados antes de 2026-07 (migración): estas columnas quedan vacías por retrocompatibilidad.

Señales de alerta

Señal Interpretación
Trazabilidad crece pero reclamos_YYYY-MM.xlsx no baja No se está cerrando ningún reclamo — problema de proceso, no de código
Más del 50% de filas son RECHAZADO El filtro COMENTARIO="reclamo" puede estar capturando demasiados falsos positivos
Mismo (MESA, MZ, LT, FECHA_COBRO) aparece dos veces Error de duplicado — investigar antes de continuar
CAMPO vacío en un reclamo RESUELTO El merge con resolucion_reclamos no encontró la fila — revisar que la clave (MESA, MZ, LT, FECHA_COBRO) coincide
