Contrato de formato — reclamos_YYYY-MM.xlsx

Módulo: 4b_reclamos  |  Ruta: outputs/reclamos_YYYY-MM.xlsx  |  Lifecycle: se regenera cada mes  |  Hojas: 1 (Reclamos)

Propósito

Vista operacional del mes activo.
Solo contiene reclamos en estado PENDIENTE y EN_REVISION.

  • El sistema auto-popula RECLAMO con el COMENTARIO que escribió el cobrador en mesa. El supervisor puede agregar detalle si revisó el archivo físico.
  • El supervisor clasifica TIPO_RECLAMO (qué componente de la deuda está disputado) y cambia ESTADO.
  • Cada re-corrida de main.py preserva las columnas manuales usando la clave (MESA, MZ, LT, FECHA_COBRO).
  • Cuando el supervisor cambia un estado a RESUELTO o RECHAZADO, la siguiente corrida mueve esa fila a trazabilidad_reclamos.xlsx y la elimina de esta vista.
  • Reclamos sin match en el mes actual (arrastres) se incluyen con MES_ANO_ORIGEN del mes en que fueron detectados.

RECLAMO (columna ámbar-morada):
El sistema lo pre-popula automáticamente con el valor del campo COMENTARIO del cobrador en mesa.
El supervisor puede agregar o editar si obtuvo información adicional revisando el archivo físico.
En re-corridas, el sistema preserva lo que el supervisor haya escrito — nunca sobreescribe.
TIPO_RECLAMO (columna ámbar):
El supervisor selecciona el componente de deuda afectado. No lo llena el cobrador.
Este valor es el que usa resolucion.py para buscar el campo correcto en DATA_boletas.
