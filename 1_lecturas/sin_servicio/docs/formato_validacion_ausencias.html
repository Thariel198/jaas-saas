Columna ESCENARIO — los dos orígenes posibles:
SIN_LECTURA — usuario presente en el template (registro_operario_mes.xlsx) pero con MARC_ACT vacío y sin obs que lo justifique · es el caso principal y el más frecuente
USUARIO_FANTASMA — usuario completamente ausente del template pero con historial en el acumulado · ocurre cuando el padrón creció y la plantilla del operario quedó desactualizada
Columna ESTADO_LISTA — qué dice la lista sobre este usuario:
NO_ESTÁ — no aparece en lista_sin_servicio.xlsx · puede ser error del operario o un caso nuevo sin registrar
SIN_AGUA — en la lista con TIPO = SIN_AGUA · tiene medidor pero sin consumo este ciclo · el sistema pre-llena DECISIÓN = sin_servicio y NOTAS con prefijo default: SIN_AGUA → sin_servicio · el supervisor puede sobrescribir si decide mantenerlo como SIN_AGUA
EN_INVEST. — ya está en la lista con TIPO = EN_INVESTIGACIÓN · el supervisor ya lo conoce pero aún no lo ha clasificado definitivamente · el sistema lo sigue incluyendo en el reporte hasta que se resuelva · tras 3 ciclos consecutivos sin lectura el sistema pre-llena DECISIÓN = sin_servicio con prefijo auto: 3 ciclos sin lectura
Nota: los SIN_MEDIDOR nunca llegan a este reporte — el sistema los acepta directamente como informativas
Columna DECISIÓN — opciones y su efecto en actualizar_lista.py:
sin_servicio — agrega el usuario a lista_sin_servicio.xlsx con TIPO = SIN_MEDIDOR y FECHA_INICIO = hoy · no volverá a aparecer en reportes futuros
error_captura — no modifica la lista · el supervisor agrega la lectura faltante al template y vuelve a correr main.py · si el usuario ahora tiene MARC_ACT la fila desaparece del reporte
investigar — agrega el usuario a la lista con TIPO = EN_INVESTIGACIÓN y FECHA_INICIO = hoy · seguirá apareciendo en reportes futuros hasta que se reclasifique · auto-promoción a SIN_MEDIDOR tras 3 ciclos consecutivos sin lectura
ignorar — no modifica la lista · queda registrado en outputs/audit_lista.xlsx con motivo ignorado · si el caso vuelve a aparecer el próximo ciclo el supervisor deberá decidir nuevamente
Bloqueante: cualquier fila con DECISIÓN vacía impide el cierre del ciclo · el sistema reporta: N ausencias sin resolver → validacion_ausencias_YYYY-MM.xlsx · las decisiones ya llenas se preservan entre ejecuciones — solo hay que llenar las que faltan
Columna MESES_SIN_LECTURA:
Ciclos consecutivos sin MARC_ACT en registro_operario_acumulado.xlsx hasta el mes actual · se recalcula en cada ejecución de validar_ausencias.py
Columna ÚLTIMO_MES_CON_LECTURA:
Último ciclo YYYY-MM donde el usuario tenía MARC_ACT registrado · vacío si nunca tuvo lectura en el acumulado actual
