05 Cobranza — arquitectura

Módulo 5 del pipeline · Carga de pagos · estado de cobro · lista de corte · arrastre

01 Lecturas → 02 Planilla → 04 Pagos → 05 Cobranza → 05b Validación → 06 Corte → 03 Boletas
Qué hace
Cargar pagos · calcular saldo · generar estado final de cobro
Lee la planilla del mes (2_planilla), los pagos identificados (4_pagos yape + efectivo) y los blancos pendientes (shared). Cruza pagos con deudas, calcula el saldo de cada usuario y genera el estado de cobro. Produce la lista de usuarios a cortar, la trazabilidad de cada pago cargado, el resumen de recaudación y el arrastre de deuda para el ciclo siguiente.
Principio de este módulo
La planilla original de 2_planilla nunca se modifica. planilla_cobrado.xlsx es un output propio — copia enriquecida con los pagos cargados. Re-correr con los mismos inputs produce el mismo output (idempotente): si no hay pagos nuevos respecto a la trazabilidad, el módulo sale sin tocar nada.

Reglas de negocio
CICLO_COBRANZA — detección autónoma
nuevo_ciclo = max(CICLO_COBRANZA en trazabilidad_cobranza) + 1
Si no hay trazabilidad previa → CICLO_COBRANZA = 1. Si los pagos entrantes ya están en trazabilidad → salir sin modificar nada (idempotencia). CICLO_COBRANZA se retroescribe también en pagos_yape_tepago.xlsx y pagos_efectivo.xlsx.
Estados de planilla_cobrado
ESTADOCondición
CANCELADOsaldo = 0 (tolerancia ±0.005)
EXCESOsaldo < 0 — pagó de más · se registra, no se devuelve hasta reclamo
PARCIALsaldo > 0 y pagó algo
PENDIENTEsaldo > 0 y no pagó nada
Condición lista_corte
SALDO > 0 AND MES_ANTERIOR ≥ 8
S/8 = mínimo mensual (mantenimiento S/3 + mínimo agua S/5). Arrastre ≥ 8 confirma que no hubo ningún pago el mes anterior. Usuarios con deuda nueva (ARRASTRE = 0) van como PARCIAL/PENDIENTE pero no a lista_corte.
PENALIDAD = S/20 · TOTAL_A_PAGAR_CORTE = SALDO + 20
Pasos internos de main.py
1
Cargar y validar inputs
Lee planilla, yape, efectivo. Falla con mensaje claro si alguno falta. Normaliza claves (MZ, LT). Carga blancos si existen.
2
Detectar CICLO_COBRANZA
Lee outputs/trazabilidad_cobranza.xlsx. Si no existe → ciclo 1. Si existe → max(CICLO_COBRANZA) + 1. Determina las filas nuevas (no procesadas aún).
3
Cruzar pagos por (MZ, LT)
Suma MONTO_ASIGNA de yape + MONTO de efectivo. Registra MONTO_YAPE, MONTO_EFECTIVO, FECHA_PAGO.
4
Aplicar blancos pendientes
Para cada usuario con blanco pendiente: escribe monto negativo en columna BLANCO de planilla_cobrado. Marca ESTADO = aplicado y escribe MES_APLICADO en blancos_acumulados.xlsx.
5
Calcular SALDO y ESTADO
SALDO = TOTAL_A_PAGAR - MONTO_YAPE - MONTO_EFECTIVO
Asigna CANCELADO / EXCESO / PARCIAL / PENDIENTE según el saldo.
6
Generar los 6 outputs
planilla_cobrado · lista_corte (filtro SALDO>0 AND ARRASTRE≥8) · trazabilidad_cobranza · resumen_recaudacion · arrastre_deuda (SALDO>0) · arrastre_devolucion (SALDO<0).
7
Retroescritura de CICLO_COBRANZA
Escribe CICLO_COBRANZA en inputs/pagos_yape/pagos_yape_tepago.xlsx e inputs/pagos_efectivo/pagos_efectivo.xlsx.

Validación post-cobranza
validacion_planilla_cobrado.py
Script independiente que verifica la coherencia interna de los 6 outputs. Se corre después de main.py. Exit code 0 = todo OK · 1 = errores encontrados.
Si hay errores → genera outputs/validacion_errores.xlsx (una hoja por bloque).
BloqueQué verifica
1 — Re-cálculo SALDORecalcula TOTAL_A_PAGAR y SALDO desde los 9 conceptos · verifica ESTADO consistente
2 — Trazabilidad ↔ planillasum(MONTO por FUENTE) en trazabilidad == MONTO_YAPE / MONTO_EFECTIVO · huérfanos
3 — arrastre_deuda ↔ planillaSALDO > 0 · monto == SALDO · faltantes y huérfanos
3 — arrastre_devolucion ↔ planillaSALDO < 0 · monto == |SALDO| · faltantes y huérfanos (espejo)
Huérfanos en Bloque 2: pagos en mesa con MZ-LT inexistentes en la planilla. Corregir en 4_pagos/efectivo/ antes de continuar al ciclo siguiente.
