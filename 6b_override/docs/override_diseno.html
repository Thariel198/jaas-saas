6b_override — diseño de archivos

Corrección de pagos mal asignados · backup automático · auditoría completa

Cuándo usar este módulo El dinero ya cuadró en 5b_validacion. Solo hay confusión de usuario: un pago quedó asignado al MZ-LOTE equivocado. Flujo de corrección por override 1. Usuario llena overrides.xlsx → FECHA + MEDIO + MZ_ANT + LOTE_ANT + MZ_NVO + LOTE_NVO + MOTIVO 2. python main.py 3. Backup automático de los archivos antes de cualquier cambio 4. Por cada override PENDIENTE: YAPE → cambia MZ/LOTE en pagos_yape_tepago.xlsx EFECTIVO → cambia MZ/LT en pagos_efectivo.xlsx Ambos → recalcula YAPE/EFEC, TOTAL_PAGADO, SALDO, ESTADO en cobranza_final.xlsx 5. Registra en trazabilidad_overrides_YYYY-MM.xlsx 6. Marca override como "aplicado" en overrides.xlsx
overrides.xlsx — archivo de entrada (el usuario llena esto)
Historia: ¿Cuándo fue el pago? → ¿Por qué canal? → ¿Dónde está mal asignado? → ¿Dónde debería ir? → ¿Por qué? → ¿Ya se aplicó?

FECHA: copiar exacto desde pagos_yape_tepago.xlsx — se usa para encontrar la fila correcta. Solo para YAPE.
Para EFECTIVO: la FECHA no es necesaria — se busca por MZ + LOTE directamente.
MEDIO: escribir exactamente YAPE o EFECTIVO.
ESTADO: no editar — lo actualiza main.py automáticamente a "aplicado".

Archivo acumulativo mensual — se crea automáticamente y se va llenando con cada ejecución.
Si ya existe del mes actual, las nuevas filas se agregan al final (no se sobreescribe).
FECHA_PAGO para EFECTIVO aparece como "—" porque pagos_efectivo no tiene columna de fecha.
backup/ — respaldos automáticos antes de cada cambio
Archivos respaldados con timestamp antes de modificar
backup/pagos_yape_tepago_2026-05-09_104512.xlsx
backup/pagos_efectivo_2026-05-09_104512.xlsx
backup/cobranza_final_2026-05-09_104512.xlsx
El backup se genera una sola vez por ejecución (no por cada override), antes del primer cambio.
Si main.py se corre dos veces en el mismo día, habrá dos timestamps distintos — ambos son válidos.
Para revertir: copiar el archivo de backup al destino original y correr main.py con los overrides desmarcados.
