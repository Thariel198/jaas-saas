discrepancias.xlsx — contrato visual

Archivo temporal · 2 hojas (discrepancias · pago_multi_mesa) · desaparece cuando ambas hojas están completamente resueltas

Una sola puerta para el supervisor — todos los pendientes aquí:
· Hoja discrepancias — 2 o más cobradores de la misma mesa registraron montos distintos para el mismo predio (MZ+LT).
· Hoja pago_multi_mesa — el mismo predio (MZ+LT) aparece cobrado en mesas distintas, riesgo de doble cobro.

El supervisor abre este archivo, llena la columna de resolución en cada hoja, guarda, cierra y vuelve a correr main.py.
· Hoja discrepancias: columna RESOLUCION (acepta_a / acepta_b / corrige).
· Hoja pago_multi_mesa: columna OK (si / rechaza / dejar vacío = pendiente).
Las resoluciones quedan en trazabilidad/trazabilidad_{mes}.xlsx (hojas discrepancias_resueltas y pago_multi_mesa_resueltas). Cuando ambas hojas están completamente resueltas el archivo se elimina automáticamente.

Valores válidos de RESOLUCION — hoja discrepancias

acepta_a / acepta_b / acepta_c Se toma el monto del cobrador indicado como correcto. Usar acepta_a, acepta_b o acepta_c según cuál cobrador tenía razón.
corrige Ningún cobrador tuvo el monto correcto. Escribir corrige en RESOLUCION y el monto real en MONTO_CORRECTO.

Hoja 2 — pago_multi_mesa

Mismo predio (MZ+LT) registrado en mesas distintas. Una fila por cada aparición. El supervisor resuelve cada fila de forma independiente: si acepta el cobro, rechaza lo descarta (queda en trazabilidad para auditoría), vacío = pendiente. Si el predio cobrado estaba mal registrado se corrige con MZ_CORRECTO / LT_CORRECTO. Múltiples si en el mismo grupo están permitidos (doble pago legítimo).

Columna OK — valores válidos — hoja pago_multi_mesa

si El cobro es válido. Entra a pagos_efectivo.xlsx.
Si MZ_CORRECTO / LT_CORRECTO están llenos, el cobro se imputa al predio corregido.
Si están vacíos, se usa el MZ+LT original.
Múltiples si en el mismo grupo están permitidos (doble pago legítimo).
rechaza El cobro se descarta — no entra a pagos_efectivo.xlsx.
Se registra en trazabilidad/pago_multi_mesa_resueltas con RESOLUCION = rechaza.
Obligatorio para cerrar el módulo: una fila sin si ni rechaza bloquea el cierre.
Vacío (pendiente) La fila sigue en discrepancias.xlsx al próximo ciclo. El archivo no desaparece mientras haya filas sin resolver.
MZ_CORRECTO / LT_CORRECTO Solo se llenan cuando el cobro se imputa a un predio distinto al original.
Ejemplo: cobrador registró MZ B LT 4 pero el predio real es MZ B LT 4A → poner B / 4A.
Si el predio original es correcto, dejar vacíos.

Reglas de comportamiento

Ciclo de resolución 1. Abrir discrepancias.xlsx
2. Hoja discrepancias: llenar RESOLUCION en cada fila
3. Hoja pago_multi_mesa: marcar acepta por grupo
4. Guardar y cerrar Excel
5. Volver a correr main.py
6. El archivo desaparece si ambas hojas están resueltas
COBRADOR_C / MONTO_C vacíos Si la mesa solo tenía 2 hojas, las columnas C salen vacías (—). No es un error — significa que no hubo 3.er cobrador.
No borrar filas El código busca exactamente las filas que generó. Si se borra una fila, el predio queda sin resolver y el archivo no desaparece.
Lifecycle Archivo temporal — nunca va a trazabilidad/. Las resoluciones quedan en trazabilidad_{mes}.xlsx hojas discrepancias_resueltas y pago_multi_mesa_resueltas.
