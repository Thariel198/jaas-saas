4_pagos / efectivo — arquitectura

Módulo de consolidación de cobros en efectivo · hasta 7 mesas · cross-check por mesa

01 Lecturas → 02 Planilla → 03 Boletas → 04 Pagos · efectivo → 05 Cobranza → 06 Corte → 07 Cierre
Qué hace
Consolidar registros de cobradores en campo → lista limpia de pagos confirmados
Lee los archivos de cada mesa (1 archivo = 1 mesa, hasta 3 hojas por archivo). Compara los registros dentro de cada mesa para detectar coincidencias o discrepancias. Entrega pagos_efectivo.xlsx listo para 5_cobranza.
Principio de este módulo
Solo consolida y valida registros manuales. No calcula si el monto es correcto (eso es 2_planilla). No cruza con Yape (eso es 4_pagos/yape). No decide si el usuario está al día (eso es 5_cobranza). Si una mesa tiene solo 1 hoja → acepta sin comparar, no bloquea.
Inputs
Manual — cobrador llena en campo Sagrado — nunca borrar sin backup
inputs/mesa_N.xlsx  (N = 1 … 7)
Un archivo por mesa física. Cada archivo puede tener 1, 2 o 3 hojas llamadas registro_1, registro_2, registro_3. Las hojas extra son opcionales — representan al 2.° y 3.° cobrador de la mesa.

Generación de templates: correr crear_templates.py una vez para crear los archivos mesa_1.xlsx … mesa_7.xlsx vacíos con el formato correcto. Los cobradores los llenan antes de correr main.py.
Reglas de negocio
1 hoja en la mesa → solo_un_cobrador
Se acepta el registro como verdad sin comparar. No bloquea el proceso. Se traza en trazabilidad/incidencias_YYYY-MM.xlsx como alerta de auditoría.
2-3 hojas coinciden en MZ + LT + MONTO → confirmado
La fila pasa directamente a pagos_efectivo.xlsx. El campo COBRADOR queda con el nombre de la primera hoja.
2-3 hojas con diferencias → discrepancia
Va a discrepancias.xlsx. El módulo termina con advertencia. Para resolver: llenar columna RESOLUCION en cada fila y volver a correr.
Regla de mayoría: 2 de 3 hojas coinciden
La fila de la mayoría pasa a pagos_efectivo.xlsx con estado mayoria_aplicada. La fila de la minoría va a trazabilidad/incidencias_YYYY-MM.xlsx para auditoría.
Mismo MZ + LT en mesas distintas → pago_multi_mesa
Ambas filas se marcan y van a trazabilidad/incidencias_YYYY-MM.xlsx. No pasan a pagos_efectivo.xlsx automáticamente — requieren revisión manual.
Pasos internos de main.py
1
Validar inputs
Verifica que exista al menos un mesa_N.xlsx en inputs/. Para cada archivo verifica que tenga al menos la hoja registro_1 con las columnas requeridas.
2
Cross-check por mesa
Para cada mesa_N.xlsx: lee todas las hojas disponibles. Si hay 1 hoja → solo_un_cobrador. Si hay 2-3 → compara por MZ+LT+MONTO y clasifica cada fila como confirmado, discrepancia o mayoria_aplicada.
3
Detectar pago_multi_mesa
Sobre el consolidado de todas las mesas, busca el mismo MZ+LT en archivos distintos. Marca esas filas y las separa para trazabilidad.
4
Escribir outputs
pagos_efectivo.xlsx ← confirmado + solo_un_cobrador + mayoria_aplicada
discrepancias.xlsx ← solo si hay discrepancias sin resolver
trazabilidad/consolidado_YYYY-MM.xlsx ← todo
trazabilidad/incidencias_YYYY-MM.xlsx ← anomalías

Lo que este módulo NO hace
✗ No calcula si el monto cobrado es correcto (eso es 2_planilla)
✗ No cruza datos con pagos Yape (eso es 4_pagos/yape)
✗ No decide si el usuario está al día (eso es 5_cobranza)
✗ No fusiona mesas distintas automáticamente — cada mesa es independiente por diseño
✗ No borra inputs/mesa_N.xlsx — son trabajo manual sagrado
