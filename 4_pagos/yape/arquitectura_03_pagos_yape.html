Prioridad de identificación
1.
Corrección acumulada — ciclo 2+ · prioridad máxima
2.
Múltiples en mensaje — "MzE Lt7, MzP Lt11A" · automático
3.
Mensaje simple — regex extrae 1 MZ-LOTE · automático
4.
Maestro ambiguo — elige menor diff · automático · va a trazabilidad
5.
Maestro único — 1 solo lote · automático
P.
Sin identificar → pendientes.xlsx · tú completas MZ/LOTE o escribes BLANCO
Ciclo 1 — mes nuevo
pagos_yape.xlsx NO existe → limpia Correcciones/ Outputs/ · Trazabilidad/ y Blancos/ intactos
Ciclo 2+ — correcciones
pagos_yape.xlsx SÍ existe → lee correcciones · actualiza trazabilidad · regenera pendientes
maestro_yape.xlsx → Outputs/ de construir_maestro → tú lo copias a shared/ → motor_matching lo lee desde shared/
⚠ Override NO vive aquí. Cuando el matching termina y el dinero cuadra en validación (04b), las correcciones de usuario por confusión de MZ-LOTE se hacen en 6b_override/ — después de corte y antes de boletas. Motor_matching solo identifica pagos, no corrige asignaciones post-validación.
Flujo mensual
1. Copiar inputs → 2. python main.py → Ciclo 1 → 3. Completar Sin_identificar → 4. python main.py → Ciclo 2 → 5. Repetir hasta pendientes = 0 → 6. pagos_yape.xlsx listo ✓ → 7. → 5_cobranza
