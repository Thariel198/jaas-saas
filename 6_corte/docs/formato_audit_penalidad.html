Propósito

Registro de auditoría de penalidades — fuente de verdad de cargos aplicados y revertidos.

  • Cada vez que aplicar_penalidad.py aplica o revierte un cargo de S/20 en planilla_mes, agrega una fila aquí.
  • Columna ACCION: APLICADO (suma +S/20) o REVERTIDO (descuenta −S/20).
  • El net activo para un (MZ, LT, MES_ANO) = filas APLICADO − filas REVERTIDO.
  • El script usa este net para la reconciliación bidireccional: aplica los que faltan, revierte los sobrantes, skipea los correctos.
  • Este archivo nunca se modifica manualmente. Es el audit trail.
Retrocompatibilidad: filas anteriores a v2.0 no tienen columna ACCION. El código las trata como ACCION=APLICADO por defecto.

Reglas de la columna ACCION

ACCION Cuándo se escribe Efecto en planilla_mes PENALIDAD_APLICADA
APLICADO Usuario está en SET_DEBE pero no en SET_TIENE — cargo nuevo CORTE_RECONEXION += 20 +S/ 20.00
REVERTIDO Usuario está en SET_TIENE pero no en SET_DEBE — cargo sobrante CORTE_RECONEXION -= 20 −S/ 20.00

Lógica de reconciliación bidireccional

SET_DEBE = { (MZ, LT) con EJECUTAR_CORTE=SI en lista_corte.xlsx }
SET_TIENE = { (MZ, LT) con net APLICADO−REVERTIDO > 0 en este audit para MES_ANO }

DEBE − TIENE → aplica +S/20 · escribe fila APLICADO
TIENE − DEBE → revierte −S/20 · escribe fila REVERTIDO
DEBE ∩ TIENE → skip — ya está correcto, sin escritura
