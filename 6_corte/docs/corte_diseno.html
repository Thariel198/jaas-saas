PAGO_NUEVO = total pagado en pagos_yape_tepago actualizado − lo que ya estaba registrado en cobranza_final.
SALDO_FINAL = SALDO − (PAGO_NUEVO − PENALIDAD). Si negativo → EXCESO (pagó de más, se le reconoce el mes siguiente en DEVOLUCION).

RECONEXION = S/40 fijo (perdió la ventana de S/20 — reconectar cuesta el doble).
TOTAL_RECONEXION = SALDO + S/40. Es lo que debe pagar para que el operario reconecte.
PAGO_EN_VENTANA: si pagó algo pero no alcanzó S/20, se muestra aquí. El pago se abona al SALDO en la planilla del mes siguiente.
Este archivo va directo al operario para que ejecute el corte físico.

Archivo persistente — se actualiza cada mes:
· Agregar: cada corte_fisico.xlsx nuevo se añade como filas CORTADO.
· Reconectar: cuando un usuario paga CORTE_RECONEXION en su planilla del mes siguiente, se marca RECONECTADO y se rellena MES_RECONEXION.
· TOTAL_RECONEXION = SALDO_AL_CORTE + S/40. Ese monto va como CORTE_RECONEXION en la planilla del mes siguiente.
· Los RECONECTADO se conservan como historial — no se eliminan.
Lógica de clasificación: PAGO_NUEVO = pagos_yape_tepago_actualizado[key] + pagos_efectivo_actualizado[key] − cobranza_final[key].yape − cobranza_final[key].efectivo.
Si PAGO_NUEVO ≥ PENALIDAD (S/20) → pagaron_penalidad · si no → corte_fisico + corte_acumulado.
